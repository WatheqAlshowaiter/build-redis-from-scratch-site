لو قرات المقالات كلها ووصلت لغاية النقطة دي ف مبروك عليك كتابة الـ InMemory database. واتمني ان المقالات كانت ممتعة كفايه بالنسبالك وواضحة

الهدف من كتابة المقالات دي كان اني اشارك المعرفة و ابسط المعلومات اللي ممكن تبقي صعبه او مش واضحة كفايه لاي حد بيحاول يفهم الـ Database internals
وازاي بتشتغل و ازاي ممكن اعمل حاجه زيها.

هتلاقي السورس كود علي [Github](https://github.com/ahmedash95/build-redis-from-scratch) و الـ commits هتلاقيها مفصلة و موضحة و متقسمه حسب اجزاء المقالة.

### اتحرك ازاي بعد كدا

في المقالات احنا كتبنا عدد قليل من الـ commands اللي Redis بيقدمها. ف تقدر انك تشوف [commands](https://redis.io/commands/) تانيه وتعملها بنفسك.
احنا بردو معملناش دعم للـ [Pipelines](https://redis.io/docs/manual/pipelining/) و ازاي نعمل
batching للـ commands. او الـ [Pub/Sub](https://redis.io/docs/manual/pubsub/) و ازاي نعمل اشتراك للـ channels و نبعت
messages للـ subscribers. او الـ.
كل دي مواضيع تقدر تعملها بنفسك و تتعلم منها.


او انك تبدا تقرا ازاي MySQL بتشتغل و علشان تسهل علي نفسك فـ [SQLite](https://sqlite.org/index.html) تعتبر simple sql database تقدر تقرا السورس كود بتاعها
و تبدا تفهم الـ concepts المختلفه عن اللي احنا كتبناه و ازاي الكلام ده بيشتغل.
